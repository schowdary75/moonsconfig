# Contributor recognition

MooNsConfig grows through focused contributions from people with different backgrounds: travel
operators, designers, documentation writers, testers, accessibility advocates, and developers.
The recognition program makes that work visible without turning contribution into a popularity
contest.

## Badge path

| Badge                | Qualification                                                            |
| -------------------- | ------------------------------------------------------------------------ |
| **Community Scout**  | An accepted issue report is resolved                                     |
| **First Flight**     | 1 merged pull request                                                    |
| **Route Builder**    | 3 merged pull requests                                                   |
| **Navigator**        | 5 merged pull requests                                                   |
| **Pathfinder**       | 10 merged pull requests                                                  |
| **Roadmap Champion** | A merged pull request completes an issue attached to a project milestone |

Badge tiers are cumulative. Roadmap Champion is a special distinction and can appear alongside any
contribution tier.

## Leaderboard scoring

The [leaderboard](LEADERBOARD.md) uses transparent, repository-derived signals:

| Contribution                                 |         Points |
| -------------------------------------------- | -------------: |
| Merged pull request                          |             10 |
| Roadmap milestone completion                 | 5 bonus points |
| Accepted issue report that is later resolved |              3 |
| Commit attributed on the default branch      |              1 |

Issues marked `duplicate`, `invalid`, or `wontfix` are not scored. Automated accounts and
maintainers are excluded from the community ranking. Maintainers are recognized separately in the
[Hall of Fame](HALL_OF_FAME.md), so their routine repository work does not crowd out community
members.

The ranking is a friendly activity summary, not a measure of personal worth or code quality.
Maintainers may correct attribution when GitHub cannot associate a contribution with an account.

## Roadmap Champions

A Roadmap Champion closes a meaningful piece of coordinated project work. Automatic recognition
requires all of the following:

1. The pull request is merged.
2. Its description uses a GitHub closing reference such as `Closes #123`.
3. The referenced issue belongs to a milestone when the pull request is merged.

Maintainers can also approve a manual Roadmap Champion nomination when work was split across
several pull requests, completed through design or research, or cannot be represented accurately
by GitHub's linking model.

## Hall of Fame

The Hall of Fame is intentionally curated. It highlights first contributions, sustained work,
milestone completions, community support, and project stewardship with a short explanation of the
impact. A leaderboard position alone does not automatically create a Hall of Fame entry.

Anyone may submit a
[recognition nomination](https://github.com/schowdary75/moonsconfig/issues/new?template=recognition_nomination.yml).
Self-nominations are welcome when they include verifiable links.

## Digital certificates

Maintainers may issue a certificate for a Roadmap Champion, Pathfinder, sustained community
support, or another clearly documented contribution. Certificates are committed as SVG files and
record the contributor, recognition, reference, date, and unique certificate ID.

See [Digital certificates](CERTIFICATES.md) for issuance and verification.

## Fairness and corrections

- Recognition is based only on public repository activity.
- No private profile information is collected.
- Documentation, tests, accessibility, design, and issue investigation are eligible.
- Contributions remain governed by the MIT License and project contribution guidelines.
- Badges and certificates are acknowledgements, not employment, academic, licensing, or
  professional credentials.
- Attribution corrections can be requested through a recognition nomination.
